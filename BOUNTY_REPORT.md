# BUG BOUNTY REPORT: CVE-2025-38352 (KGSL OOB Read/Write)

## Executive Summary

A critical vulnerability exists in the `KGSL` (Kernel Graphics Support Layer) driver specifically on the Honor Magic V2 and other Snapdragon 8 Gen 2 devices. The flaw allows a process with `shell` (UID 2000) privileges to trigger an out-of-bounds (OOB) memory access, leading to arbitrary kernel memory read and write capabilities.

## Technical Details

- **Vulnerability Type**: Out-of-bounds Read/Write (CWE-787)
- **Affected Component**: `/dev/kgsl-3d0`
- **Root Cause**: An integer overflow or type confusion in the `IOCTL_KGSL_GPUOBJ_ALLOC` and `IOCTL_KGSL_GPUOBJ_IMPORT` handlers allows for bypassing alignment checks when `PAGE_SIZE` vs `64KB` alignment is enforced.
- **Exploitation Path**:
    1. A malicious app (or shell user) triggers a structural mutation in the GPU object list.
    2. By mapping specifically crafted GPU addresses, the user gains a "dangling" reference.
    3. Memory reclamation (Slab-reclaim) allows hijacking a `pipe_buffer` or `msg_msg` struct.
    4. KASLR is bypassed via a separate info-leak in `DEVICE_GETPROPERTY`.

## Impact

- **Confidentiality**: High (Full kernel memory read)
- **Integrity**: High (Full kernel memory write)
- **Availability**: High (System crash/REPL)
- **CVSS Score**: 7.8 (High)

## Forensic Evidence (Sovereign Guard Logs)

We have observed that while the **OOB Read/Write** is fully achievable, attempts to modify sensitive kernel structures (like `cred->uid`) are intercepted by the **EL2 Hypervisor (MagicGuard / RKP)**. The hypervisor nullifies pointers (`0x1`) when specific mutation patterns are detected.

## Mitigation Suggestions

1. **Strict Input Validation**: Enforce 64KB alignment for all GPU object imports.
2. **Access Control**: Restrict KGSL IOCTL access for untrusted applications and ensure `shell` does not have excessive GPU debug permissions.
3. **Hypervisor Hardening**: Continue enhancing the EL2 RKP to protect the `task_struct` and `cred` lists.

## Steps to Reproduce

1. Build the provided `src/kread_dump.c`.
2. Push to `/data/local/tmp/`.
3. Execute via ADB.
